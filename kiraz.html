Error Pages

For an e-commerce website builder SaaS, the project involved designing error pages. I did my best to stick with these design choices, which I got great praise for at the end of the project, because they met the needs of the company and the brand identity, which is playful, lively, and circular.

404 Page

For 404 pages, I provide the user with a title and an explanation before adding a search bar to aid in finding what they need because they couldn't find their way around. I added the Home primary CTA button and the Back secondary CTA button below the search bar to make it easier for users to navigate.

503 Page

Since there are only two options available to users for 503 errors—refresh the page or leave—I only provide an explanation and a refresh button.

Mobile Versions

Since there are only two options available to users for 500 errors—refresh the page or leave—I only provide an explanation and a refresh button.

Mobile Version

Since this application will be primarily used on mobile devices, I also created mobile versions of the pages. I continue to use responsive design guidelines.
